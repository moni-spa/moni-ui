# Chips

Chips help people enter information, make selections, filter content, or trigger actions

![4 chip variants: Assist, filter, input, and suggestion.](../../assets/chips/guidelines/01-4-chip-variants-assist-filter-input-and-suggestion-9c99ba32.png)

There are four variants of chips: Assist, filter, input, and suggestion

## Usage

Chips help people enter information, make selections, filter content, or trigger actions. They're best used to help people accomplish their current task faster and easier.

![3 assist chips in restaurant review app.](../../assets/chips/guidelines/02-3-assist-chips-in-restaurant-review-app-b707422e.png)

Chips appear as a group of interactive elements

### Chips aren’t buttons

Chips and buttons [More on buttons](/m3/pages/common-buttons/overview) are similar. They both provide visual cues to prompt people to take actions and make selections. While buttons appear consistently and with familiar calls to action, chips are dynamic to the situation, and appear as a group of interactive elements. Use chips to enhance a person's current journey and encourage action. Use buttons to progress them through the product and for significant actions.

![Chips used for “Reserve a table” and “Order delivery” in a restaurant app.](../../assets/chips/guidelines/03-chips-used-for-reserve-a-table-and-order-delivery-in-a-resta-b764de80.png)

check Do

Use chips to present contextual, supplemental options

![Chips used for Save and Cancel actions in a dialog.](../../assets/chips/guidelines/04-chips-used-for-save-and-cancel-actions-in-a-dialog-236927ef.png)

close Don’t

Avoid replacing major actions with chips. Actions that progress people to the next or previous step should always be displayed as buttons. Chips should dynamically offer various actions depending on the current task, whereas a button should be a persistent fixture of a layout. Chips represent forking paths for a current task, while buttons represent linear steps.

![Button labeled “Save task” in a task management app.](../../assets/chips/guidelines/05-button-labeled-save-task-in-a-task-management-app-3fa13850.png)

check Do

Use buttons for the final step in a task

![Chip labeled “Save task” in a task management app.](../../assets/chips/guidelines/06-chip-labeled-save-task-in-a-task-management-app-81c9295a.png)

close Don’t

Avoid using chips to finish or progress a task

Multiple chips should appear together in a set, whereas there should be no more than 3 buttons in a single arrangement. Chip sets can be scrolled horizontally.

![Multiple chips in event reminder.](../../assets/chips/guidelines/07-multiple-chips-in-event-reminder-66a86c81.png)

check Do

Chips can be scrolled horizontally

![Single chip in event reminder.](../../assets/chips/guidelines/08-single-chip-in-event-reminder-756252a3.png)

close Don’t

Don’t display a single chip by itself. Chips should appear in a set.

## Anatomy

![Diagram of 4 elements of a chip.](../../assets/chips/guidelines/10-diagram-of-4-elements-of-a-chip-fea11e6d.png)

1. Container
2. Label text
3. Leading icon or image (optional)
4. Trailing icon (required for input chips, optional for filter chips)

### Container

All chips are slightly rounded with an 8dp corner.

![Diagram of rounded corner of chip container.](../../assets/chips/guidelines/11-diagram-of-rounded-corner-of-chip-container-14dcf6e1.png)

Chips have rounded corners

When on complicated backgrounds, chip containers can be elevated

### Label text

Chip label text should be 20 characters or fewer, and have the same typography style as buttons [More on buttons](/m3/pages/common-buttons/overview). Chip labels should remain brief for the limited space available. Skip conventional grammar rules, such as articles (take "a" walk), to save space.

![Filter chip in maps app labeled “Walking.”](../../assets/chips/guidelines/17-filter-chip-in-maps-app-labeled-walking-8b644d92.png)

check Do

Keep chip labels short

![Filter chip in maps app labeled “Walking navigation session."](../../assets/chips/guidelines/18-filter-chip-in-maps-app-labeled-walking-navigation-session-b35442de.png)

close Don’t

Avoid chip labels longer than 20 characters

### Leading icon or image (optional)

Chips can contain a leading icon, logo, or circular image. Use a system icon to help identify a chip's category.

![2 input chips, 1 with a leading circular avatar and the other with a leading icon.](../../assets/chips/guidelines/19-2-input-chips-1-with-a-leading-circular-avatar-and-the-other-5c5d303b.png)

Chips can contain a logo, icon, or circular image

The leading icon color for unselected chips can be customized through theming. While the default color role [More on color roles](/m3/pages/color-roles) is **primary**, the **on surface variant** color role is a good alternative when the icon style requires less emphasis.

![2 screens of a restaurant page with chips labeled "Add photos" and "Add dishes." The leading icons are purple in 1, grey in the other.](../../assets/chips/guidelines/20-2-screens-of-a-restaurant-page-with-chips-labeled-add-photos-d6da70ce.png)

**Primary** color (left) and **on surface variant** color (right)

Leading circular images are sized larger than leading icons to provide more space for detail. Icons are designed to be legible at small sizes. See the [Specs tab](/m3/pages/chips/specs/) for precise measurement values.

![2 input chips, 1 with a 24dp leading avatar and the other with an 18dp leading icon.](../../assets/chips/guidelines/21-2-input-chips-1-with-a-24dp-leading-avatar-and-the-other-wit-070f98a5.png)

Leading images, such as avatars, are sized larger than leading icons or logos

### Trailing icon (optional, input and filter chips only)

The trailing icon is present for input and filter chips . On input chips, it's required and must be used to remove the chip. On filter chips, it's optional, and can be used to open a menu or remove the chip. Secondary actions (such as a trailing icon button for **Remove**) must have a 48x48dp interaction target that doesn’t interfere with the chip's primary action (such as **Edit** or **Drag**). To achieve this, apply a minimum width of 88dp to the chip, or 42dp to the label text.

![A chip with 88dp min width has 2 touch targets of 48x48dp.](../../assets/chips/guidelines/22-a-chip-with-88dp-min-width-has-2-touch-targets-of-48x48dp-3f451853.png)

Interaction targets for actions like **Edit** or have a size of 48x48. This can be achieved by setting the minimum container width to 88dp.

## Assist chips

Assist chips represent smart or automated actions that can span multiple apps, such as opening a calendar event from the home screen. Assist chips function as though the person asked an assistant to complete the action. They should appear dynamically and contextually in a UI. An alternative to assist chips are buttons [More on buttons](/m3/pages/common-buttons/overview), which should appear persistently and consistently.

![2 assist chips labeled “Add to my itinerary” and “12 mins from hotel” on map screen about Los Cantantes.](../../assets/chips/guidelines/23-2-assist-chips-labeled-add-to-my-itinerary-and-12-mins-from--2742ed81.png)

The text in most assist chips begins with a short verb, like **Get** or **Add**

![2 assist chips labeled “Event location” and “Chat” on phone lock screen.](../../assets/chips/guidelines/24-2-assist-chips-labeled-event-location-and-chat-on-phone-lock-159af4f0.png)

An assist chip can surface supplemental information like a calendar event, as well as provide contextual actions

![Assist chip transformed into time picker with radio buttons.](../../assets/chips/guidelines/25-assist-chip-transformed-into-time-picker-with-radio-buttons-eef90422.png)

During an interaction, assist chips can transform into modals, transition into full-screen views of new content, or readjust to display more results inline

Assist chips can trigger an action or show progress and confirmation. Write assist chips like buttons: start with a verb. Adjust text dynamically if the state changes, like **Save** to **Saved**. Tapping an assist chip triggers a contextual action

Assist chips can show progress and confirmation feedback

Assist chips are displayed after primary content, such as below a card [More on cards](/m3/pages/cards/overview) or persistently at the bottom of a screen.

![3 assist chips labeled “Turn on lights”, “Set alarm”, and "Close Blinds" on Welcome Home screen.](../../assets/chips/guidelines/26-3-assist-chips-labeled-turn-on-lights-set-alarm-and-close-bl-f0760973.png)

Assist chips should be shown underneath primary content

## Filter chips

Filter chips use tags or descriptive words to filter content. They can be a good alternative to segmented buttons [More on segmented buttons](/m3/pages/segmented-buttons/overview) or checkboxes [More on checkboxes](/m3/pages/checkbox/overview) when viewing a list or search results. Tapping on a filter chip activates it and appends a leading checkmark icon to the starting edge of the chip label. Write filter chips with nouns that describe the category to **include** in the results. Avoid negative phrases like **Exclude images**.

![4 filter chips labeled “Docs” “Slides” “Sheets” and “Images” with “Slides” and “Sheets” selected](../../assets/chips/guidelines/27-4-filter-chips-labeled-docs-slides-sheets-and-images-with-sl-fb1fbb10.png)

Filter chips rely on tags or descriptive words to filter content

![5 filter chips representing popular electronics categories in a shopping app. The chip “Game consoles” is selected.](../../assets/chips/guidelines/28-5-filter-chips-representing-popular-electronics-categories-i-5bf5d57f.png)

Filter chips in a shopping app

![6 filter chips representing apartment amenities in a real estate app. 2 of the chips are selected.](../../assets/chips/guidelines/29-6-filter-chips-representing-apartment-amenities-in-a-real-es-ef81e3bc.png)

Filter chips in a real estate app

Tap a chip to select it. Multiple chips can be selected or unselected.

![6 filter chips representing apartment amenities in a real estate app. 3 selected chips include checkmarks while 3 don’t.](../../assets/chips/guidelines/30-6-filter-chips-representing-apartment-amenities-in-a-real-es-abce66e3.png)

Filter chip suggestions can dynamically change as a person starts to select filters

Alternatively, a single chip can be selected. This offers an alternative to segmented buttons [More on segmented buttons](/m3/pages/segmented-buttons/overview), radio buttons [More on radio buttons](/m3/pages/radio-button/overview), or single select menus [More on menus](/m3/pages/menus/overview). However, avoid mixing chip set behaviors. All chip sets on a page should be either single-select or multi-select. Filter chips can be set so that selecting a single chip automatically deselects all other chips in the set

In medium [More on medium window size class](/m3/pages/applying-layout/medium) and expanded window sizes [More on expanded window size class](/m3/pages/applying-layout/expanded), filter chips may contain a trailing icon to directly remove the chip or open a menu of options. In compact windows [More on compact window size class](/m3/pages/applying-layout/compact), the trailing icon's target area is too small to be accessible on its own. However, if the whole chip can be selected to accomplish the action, the chip is likely still accessible.

## Input chips

Input chips represent discrete pieces of information entered by a person, such as Gmail contacts or filter options within a Search lets people enter a keyword or phrase to get relevant information. More on search [More on search](/m3/pages/search/overview) field. They enable user input and verify that input by converting text into chips.

![Contacts transformed into 2 input chips in a draft event invite](../../assets/chips/guidelines/39-contacts-transformed-into-2-input-chips-in-a-draft-event-inv-7ae07c9e.png)

Input chips transform text based on a person's input

Input chips can support editing to change their contents, such as correcting an email address. In edit mode, the chip reverts back to a text string. Editing can be triggered by interacting with the chip, either by selecting it or by a second interaction after selection. Input chips converted from email addresses are editable

Using the backspace key with the cursor before a chip selects the entire chip. The chip can then be deleted when the user taps the backspace key again. A single field can contain multiple input chips . These chips can be reordered or moved into other fields.

![2 input chips in the “To” field of an email draft](../../assets/chips/guidelines/40-2-input-chips-in-the-to-field-of-an-email-draft-bfaac50f.png)

Multiple input chips in one field

![1 input chip moved from the “To” field to the “CC” field of an email draft](../../assets/chips/guidelines/41-1-input-chip-moved-from-the-to-field-to-the-cc-field-of-an-e-c54ffe21.png)

Input chips being moved from one field to another

Input chips can expand to show more information or options. A container transform transition pattern is used to reveal additional content. Input chips can expand

### Placement

Input chips can be integrated with other components. They can appear:

- Inline with the cursor in a text field
- In a stacked list
- In a list that can be horizontally scrolled

![4 input chips wrapping to 2 rows.](../../assets/chips/guidelines/42-4-input-chips-wrapping-to-2-rows-91be0066.png)

Input chips can wrap to a new row if all chips need to be visible

Input chips can scroll horizontally

### Icons & images

The leading icon of input chips can be an icon, logo, or circular image.

![2 input chips, 1 with leading thumbnail, 1 with leading icon.](../../assets/chips/guidelines/43-2-input-chips-1-with-leading-thumbnail-1-with-leading-icon-211cd1ad.png)

Input chips can contain an icon, logo, or circular image

The trailing icon is always aligned to the end side of the container. It’s placed:

- On the right for left-to-right (LTR) languages, such as English
- On the left for right-to-left (RTL) languages, such as Farsi

![2 input chips with trailing icons following the label text.](../../assets/chips/guidelines/44-2-input-chips-with-trailing-icons-following-the-label-text-40ae45c5.png)

Input chips can be a more flexible way to filter search results, compared to filter chips

![3 input chips with trailing icons following the label text.](../../assets/chips/guidelines/45-3-input-chips-with-trailing-icons-following-the-label-text-9ec8809f.png)

Input chips make it easier to add and remove contacts

Overflowed chips in a text field should follow the same behavior as regular text. An unfocused text field with overflowed content should display the beginning of the input. Tapping the field snaps the user to the end of the input with the cursor and keyboard active.

## Suggestion chips

Suggestion chips help narrow a user’s intent by presenting dynamically generated suggestions, such as possible responses or search filters. Write suggestion chips as nouns or short phrases, depending on context. Avoid exceeding 20 characters when possible.

![3 suggestion chips offering ID options in a photo-recognition app.](../../assets/chips/guidelines/46-3-suggestion-chips-offering-id-options-in-a-photo-recognitio-74a64e63.png)

The text labels within suggestion chips are most often nouns or short phrases

![3 suggestion chips with automated reply options in a chat app.](../../assets/chips/guidelines/47-3-suggestion-chips-with-automated-reply-options-in-a-chat-ap-a2e717e9.png)

Suggestion chips can offer quick-reply options in a chat or email app

![2 suggestion chips with search options “What song is this?” and “What are the lyrics?”](../../assets/chips/guidelines/48-2-suggestion-chips-with-search-options-what-song-is-this-and-9dedf7ab.png)

A suggestion chip can help the user start a search

### Placement

When displaying multiple chips together, place them inline as a row of options, not listed vertically. Overflowing chips should break to the next line. If the field is only one row high, chip sets can scroll horizontally instead. Keep an 8dp minimum space between chips. Chips must also have a minimum 48dp target size, regardless of placement or density. If required, the target can extend beyond the visible container of the chip. Text labels for chips should be concise. Chip labels will truncate when in a wrapped layout, and when they are wider than the full width of the window.
